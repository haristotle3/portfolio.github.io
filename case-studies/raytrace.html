Real-Time Ray Tracing

1. Bounding Volume Hierarchy (BVH)

Google Earth employs ray tracing with BVH acceleration structures to render realistic lighting in real-time. A BVH organizes 3D objects in a tree structure where each node contains a bounding volume of its children. This enables efficient ray-scene intersection tests by:

  • Quickly eliminating large groups of non-intersecting objects
  • Reducing O(n) complexity to O(log n) for n objects
  • Optimizing for dynamic scenes through partial tree rebuilds
Google's implementation specifically optimizes for planetary-scale rendering with LOD (Level of Detail) systems.

2. Hybrid Rasterization-Ray Tracing Pipeline

Google Earth uses a hybrid approach combining traditional rasterization with selective ray tracing:

  • Rasterization for primary visibility and base geometry
  • Ray tracing for accurate shadows, reflections, and ambient occlusion
  • Screen-space denoising to reduce ray counts
This balances visual fidelity with the performance demands of streaming 3D maps.

Implementation (Simplified BVH Traversal):


struct BVHNode {
    AABB bounds;
    bool isLeaf;
    union {
        int child[2];  // Interior nodes
        int firstPrim;  // Leaf nodes
        int primCount;
    };
};

bool IntersectBVH(Ray ray, BVHNode[] nodes, Primitive[] prims, float tMax) {
    Stack stack;
    stack.push(0);  // Root node index
    
    while (!stack.empty()) {
        int nodeIdx = stack.pop();
        BVHNode node = nodes[nodeIdx];
        
        if (!IntersectAABB(ray, node.bounds, tMax)) 
            continue;
            
        if (node.isLeaf) {
            for (int i = 0; i < node.primCount; i++) {
                Primitive prim = prims[node.firstPrim + i];
                if (IntersectPrimitive(ray, prim, tMax)) 
                    return true;
            }
        } else {
            stack.push(node.child[0]);
            stack.push(node.child[1]);
        }
    }
    return false;
}

// Google Earth's optimized version uses:
// - SIMD instructions for parallel AABB tests
// - Persistent threads for coherent ray traversal
// - Instance acceleration for repeated structures

Key Insights

  • Google Earth uses ray tracing selectively for the highest visual impact
  • BVH structures enable real-time performance at global scale
  • Hybrid approaches balance quality and performance
  • Specialized for geospatial data (height fields, buildings, vegetation)
  • WebGL 2.0 implementation works across devices
