Path Planning for Self-Driving Cars

1. Vehicle Routing Problem (VRP) with Graph Optimization

The Vehicle Routing Problem (VRP) extends the Traveling Salesman Problem (TSP) to optimize routes for multiple vehicles serving numerous locations. Waymo's autonomous vehicles use a hybrid approach combining:

  • Graph representations of road networks (nodes=intersections, edges=road segments)
  • Constraint programming for traffic rules and vehicle constraints
  • Metaheuristics like Tabu Search for large-scale optimization
The objective minimizes total travel time while respecting capacity, time windows, and traffic constraints.

2. Dynamic Programming with A* Hybridization

For real-time path planning, Waymo combines:

  • Dynamic Programming: Breaks problems into overlapping subproblems (Bellman's principle)
  • A* Algorithm: Heuristic-guided search with real-time traffic data
  • Constrained Optimization: Vehicle kinematics and safety margins
This hybrid approach enables millisecond-scale replanning when encountering obstacles or traffic changes.

Implementation (VRP with Time Windows):


import numpy as np
from ortools.constraint_solver import routing_enums_pb2
from ortools.constraint_solver import pywrapcp

class VRPSolver:
    def __init__(self, num_vehicles, depot_index, time_matrix, time_windows, demands, vehicle_capacities):
        self.num_vehicles = num_vehicles
        self.depot_index = depot_index
        self.time_matrix = time_matrix
        self.time_windows = time_windows
        self.demands = demands
        self.vehicle_capacities = vehicle_capacities
        
    def solve(self):
        # Create routing model
        manager = pywrapcp.RoutingIndexManager(
            len(self.time_matrix), self.num_vehicles, self.depot_index)
        routing = pywrapcp.RoutingModel(manager)

        # Add time dimension
        def time_callback(from_index, to_index):
            from_node = manager.IndexToNode(from_index)
            to_node = manager.IndexToNode(to_index)
            return self.time_matrix[from_node][to_node]
        
        transit_callback_index = routing.RegisterTransitCallback(time_callback)
        routing.SetArcCostEvaluatorOfAllVehicles(transit_callback_index)
        
        # Add time window constraints
        time = 'Time'
        routing.AddDimension(
            transit_callback_index,
            30,  # allow waiting time
            1440,  # maximum time per vehicle (minutes)
            False,  # Don't force start cumul to zero
            time)
        time_dimension = routing.GetDimensionOrDie(time)
        
        for location_idx, window in enumerate(self.time_windows):
            if location_idx == self.depot_index:
                continue
            index = manager.NodeToIndex(location_idx)
            time_dimension.CumulVar(index).SetRange(window[0], window[1])
        
        # Add capacity constraints
        def demand_callback(from_index):
            from_node = manager.IndexToNode(from_index)
            return self.demands[from_node]
        
        demand_callback_index = routing.RegisterUnaryTransitCallback(demand_callback)
        routing.AddDimensionWithVehicleCapacity(
            demand_callback_index,
            0,  # null capacity slack
            self.vehicle_capacities,  # vehicle maximum capacities
            True,  # start cumul to zero
            'Capacity')
        
        # Solve with hybrid metaheuristic
        search_parameters = pywrapcp.DefaultRoutingSearchParameters()
        search_parameters.first_solution_strategy = (
            routing_enums_pb2.FirstSolutionStrategy.PATH_CHEAPEST_ARC)
        search_parameters.local_search_metaheuristic = (
            routing_enums_pb2.LocalSearchMetaheuristic.GUIDED_LOCAL_SEARCH)
        search_parameters.time_limit.seconds = 5  # Real-time constraint
        
        solution = routing.SolveWithParameters(search_parameters)
        return self._format_solution(manager, routing, solution)
    
    def _format_solution(self, manager, routing, solution):

        pass

# Example usage:
time_matrix = [
    [0, 6, 9, 8, 7],  # Depot to locations
    [6, 0, 5, 4, 3],
    [9, 5, 0, 2, 1],
    [8, 4, 2, 0, 1],
    [7, 3, 1, 1, 0]
]
time_windows = [(0, 1440), (5, 15), (10, 20), (5, 15), (0, 1440)]  # (open, close)
demands = [0, 1, 1, 2, 1]  # Depot has 0 demand
vehicle_capacities = [3, 3]  # Two vehicles with capacity 3

solver = VRPSolver(
    num_vehicles=2,
    depot_index=0,
    time_matrix=time_matrix,
    time_windows=time_windows,
    demands=demands,
    vehicle_capacities=vehicle_capacities
)
solution = solver.solve()

Basic VRP O(n!) Worst-case for exact solutions

Real-time A* O(bᵈ) b=branching factor, d=depth

Key Insights

  • VRP formulations can reduce fleet mileage by 15-30% compared to naive routing
  • Time windows enable efficient coordination of autonomous fleets
  • Google OR-Tools provides production-grade VRP solvers used by Waymo
  • Online algorithms continuously reroute based on real-time traffic data
  • Multi-objective optimization balances time, energy use, and passenger comfort
